Front end development

I developed strong background in Front-end development due to two years of exposure in a large React codebase with beautiful architecture. I spearheaded React initiatives across multiple teams comprising of hundred developers. My presentations included topics like "TDD with React", "Understanding scope of Unit tests and Integration tests", "Code review guidelines". I can architect a complete process for a new project starting from design, development and testing in local environment through integration, staging till deployment in production environment. Having a strong background in Nodejs gave me an advantage over others when working with frameworks like React or Angular. I consider end to end debugging skills to be my strongest suit in front end development.

Contrary to a programmer, designer has a tough job of incorporating best User experience while complying with style guidelines from Product engineering. I am a strong proponent of separate UI/UX design team. As per my experience switching between design and development in a big project is often counterproductive in the long run but it doesn't mean that developers can shy away from design decisions. Design elements like color-combinations, spacing, animations must be consistent through out the application and requires special coordination when multiple teams are working on same application suite. A comprehensive style guide is indispensable.
